# 2026-07-23 — FBNeo → WebAssembly (contrato wasm-gaming)

Port de **FinalBurn Neo** (emulador arcade multi-sistema) a WebAssembly, empaquetado
como engine SDK conforme al contrato de [`@wasm-gaming/engine-specs`](https://github.com/wasm-gaming/engine-specs),
igual que los engines hermanos (blastem-wasm, jgenesis-wasm, rsdkv*).

**Estado: FUNCIONAL de punta a punta.** Metal Slug (`mslug` + BIOS `neogeo`) carga,
renderiza, y corre en el navegador con **vídeo, sonido y a velocidad correcta (~59.18fps)**.

---

## 1. Qué se ha hecho

### Capa SDK (contrato wasm-gaming)
- `src/fbneo.sdk.ts` — `EngineSDK` (`{ manifest, load }`). `load()` escribe la ROM/BIOS
  en MEMFS y arranca FBNeo con el nombre corto del romset (`callMain([driver])`).
- `src/fbneo.manifest.ts`, `src/fbneo.options.ts` — manifest + opciones tipadas + JSON Schema.
- `src/demo/` — demo con selector de ROM/BIOS + campo "romset name".
- Typecheck en verde; el manifest valida contra el `validateManifest` del propio contrato.
- Scaffolding espejo de blastem-wasm: `Makefile`, `scripts/`, workflows CI, README, CORE.md.

### Port del emulador a WASM (`scripts/build-fbneo.sh`)
FBNeo **no tiene target oficial de Emscripten**. El script clona `finalburnneo/FBNeo`,
parchea `makefile.sdl2` y compila con emsdk vía Docker. Produce `dist/fbneo/fbneo.js`
(~306KB, loader `createFbneoModule`) + `dist/fbneo/fbneo.wasm` (~38MB, incluye todos
los drivers arcade + decoder MP3).

Comandos: `make build-wasm` (Docker), `make build-sdk` (TS), `make preview` (sirve `dist/`
con cabeceras COOP/COEP). Modo iteración rápida: `FBNEO_INCREMENTAL=1` re-enlaza en ~1-2 min.

## 2. Puntos críticos (los parches que hacen que funcione)

Todos los parches del emulador van en `scripts/build-fbneo.sh`, guardados con
`#ifdef __EMSCRIPTEN__` para no tocar el comportamiento nativo.

### Build (5 bloqueos estructurales)
1. **Makefile real**: es `makefile.sdl2` en la raíz (NO `src/burner/sdl/makefile.burn`).
2. **MAKELEVEL**: hay que invocar `make sdl2` (target del makefile raíz, que anida
   `$(MAKE) -f makefile.sdl2`). El build es multi-pasada por `MAKELEVEL`; invocarlo
   directo empieza en nivel 0 y nunca enlaza (error espurio `fbneo.o`).
3. **Toolchain**: el makefile hardcodea `CC = gcc` → sed a `emcc`/`em++`. Las
   herramientas de generación de código (m68kmake, *.pl) necesitan toolchain **nativo**
   (`HOST_CC=cc HOST_CXX=c++` + `build-essential` en el contenedor).
4. **SDL2/libs**: `sdl2-config` no existe en emsdk → `-sUSE_SDL=2 -sUSE_SDL_IMAGE=2`;
   quitar `-lGL/-lopengl32/-lSDL2_image/-lpthread`; desactivar x86 (`BUILD_X86_ASM=`
   `FASTCALL=`); forzar salida `.js`.
5. **`std::byte`**: la libc++ del emsdk lo define como templates (C++17); FBNeo incluye
   `<cstddef>` dentro de `extern "C"` → "templates must have C++ linkage". Fix:
   `-include cstddef` en CXXFLAGS (no-op guardado, conserva C++17).

### Enlace
6. **drmp3**: el `master` reciente de FBNeo usa dr_mp3 para SNES MSU1, pero la
   implementación solo se compila en Win32/libretro → símbolos indefinidos. Fix:
   activar `INCLUDE_FLACMP3_SUPPORT` también para `__EMSCRIPTEN__` en `samples.cpp`.

### Carga de ROMs
7. **TextDecoder + memoria crecible**: con `ALLOW_MEMORY_GROWTH`, el heap es un
   `ArrayBuffer` *resizable* y Chrome's `TextDecoder.decode()` lanza. `-sTEXTDECODER=0`
   NO es válido → se usa **`ALLOW_MEMORY_GROWTH=0` + `INITIAL_MEMORY=512MB` fijo**.
8. **Rutas de ROM**: el SDK escribe la ROM/BIOS en las rutas de búsqueda por defecto de
   FBNeo (`drv.cpp`): la absoluta `/usr/local/share/roms/` **y** `/roms/`. El zip debe
   llamarse `<driver>.zip`.
9. **⭐ `%hs` (el bug estrella)**: la libc **musl** de Emscripten **no implementa** el
   especificador `%hs` (narrow-string de MSVC) → lo expande a vacío. FBNeo construye las
   rutas de ROM con `_stprintf("%s%hs", ruta, nombre)`, así que el nombre desaparecía →
   buscaba `/usr/local/share/roms/.zip` → **todo "not found"** (incl. BIOS). Fix:
   `sed 's/%hs/%s/g'` en `src/burner/sdl/*.cpp` (aquí `TCHAR=char`, así que `%hs`≡`%s`).

### Runtime (velocidad, cuelgue, sonido)
10. **Cuelgue de pestaña**: `RunMessageLoop` es un `while(!quit)` bloqueante → nunca
    devuelve el control al navegador. Con ASYNCIFY se cede cada frame con
    `emscripten_sleep`. Además se desactiva el auto-pausado por `SDL_WINDOWEVENT_FOCUS_LOST`
    (el canvas no tiene foco de teclado al arrancar → salía "PAUSE").
11. **Velocidad ×100**: `GetTime()` (run.cpp) devuelve **microsegundos**, pero la fórmula
    de pacing de `RunIdle` (`nCount = nTime * nAppVirtualFps / 100000`) espera **ms**. En
    nativo ese camino está muerto (usa sync por audio), por eso el bug estaba latente.
    Fix: devolver ms bajo emscripten.
12. **Velocidad ×2.17 + silencio (audio)**: el `AudioContext` del navegador es nativo
    **F32/48000**; FBNeo pedía **S16/44100**. El callback avanzaba el ring buffer S16 por
    bytes-F32 (×2) + desajuste 48000/44100 (×1.088). Fix: por defecto core a **48000**
    (`aud_interface.cpp`), pedir `AUDIO_F32`, abrir con `SDL_OpenAudioDevice(...,
    allowed_changes=0)`, y **convertir S16→float en el callback** avanzando `nSDLPlayPos`
    en unidades S16 (`aud_sdl.cpp`). Forzar S16 daba velocidad correcta pero **silencio**
    (Web Audio necesita float nativo). Con esto el pacer por audio se auto-sincroniza a
    ~59.18fps con sonido real.

### Técnica de depuración clave
El arnés en Node (`.tmp/harness.cjs`) **no puede** ejecutar el bucle de juego (SDL
vídeo/audio fallan headless). Para diagnosticar velocidad/audio se **instrumenta el wasm**
(printf de fps/`bAudPlaying` en `RunIdle`, y del `audiospec` obtenido en `SDLSoundInit`) y
se lee la **consola del navegador real**. Así se descubrió el mismatch F32/48000
(`format=0x8120=AUDIO_F32`) y el bug de `%hs` (instrumentando `FileExists`/`BurnGetZipName`).

## 3. Lo que queda por hacer

### Pendiente / pulido (no bloqueante)
- **Audio asume dispositivo a 48000Hz** (lo típico en navegadores). En un dispositivo a
  44100/96000 habría ~8% de desajuste de pitch/velocidad. Robusto sería leer
  `audiospec.freq` obtenido y ajustar `nAudSampleRate` antes de init del core (problema de
  orden de inicialización), o resampling dinámico como jgenesis.
- **`ScriptProcessorNode` deprecado**: emscripten SDL2 lo usa; funciona pero la API
  moderna es `AudioWorkletNode`. Mejora futura.
- **GL "immediate mode emulation"**: FBNeo usa OpenGL clásico que Emscripten emula de forma
  limitada (`LEGACY_GL_EMULATION`). Funciona para NeoGeo/CPS; algunos drivers con efectos
  GL avanzados podrían fallar. Alternativa: ruta de vídeo del renderer SDL2 (`vid_sdl2.o`).
- **Input**: `setInput()` del SDK es no-op (el input va por el event loop SDL2 con el mapeo
  por defecto: moneda `5`, start `1`, flechas, disparo `Z/X/C/V`). Falta mapear el contrato
  de input de wasm-gaming a los bindings de FBNeo.
- **Capabilities del manifest**: `saveStates`/`sram` están en `false`. FBNeo los soporta;
  falta exponer `saveState()`/`loadState()` del SDK contra la API del core y persistencia
  (IDBFS/OPFS).
- **Verificar más juegos**: solo probado mslug (NeoGeo). Conviene probar CPS-1/2/3, sets con
  BIOS distinta, y juegos verticales (geometría de vídeo).
